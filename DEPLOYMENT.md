# Инструкция по локальному развёртыванию Health Tracker

## Описание проекта

Health Tracker — фронтенд-прототип мобильного приложения для персонализированного трекера здоровья и питания. Включает 4 экрана:
- **Дашборд** — сводка по калориям, макронутриентам и дефицитам
- **Чат** — ввод данных о питании и активности через чат
- **Продукты и рецепты** — список продуктов/рецептов для закрытия дефицитов
- **Список покупок** — редактируемый список покупок на неделю

Все данные загружаются из мок-объектов (структура соответствует будущему API).

## Требования

- **Node.js** версии 18 или выше ([скачать](https://nodejs.org/))
- **pnpm** версии 8 или выше

### Установка pnpm (если не установлен)

```bash
npm install -g pnpm
```

---

## Шаги по развёртыванию

### 1. Клонирование репозитория

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd <ПАПКА_ПРОЕКТА>
```

### 2. Установка зависимостей

Из корня проекта выполните:

```bash
pnpm install
```

### 3. Запуск приложения

```bash
pnpm --filter @workspace/health-tracker run dev
```

После запуска приложение будет доступно по адресу:

```
http://localhost:24166/
```

Откройте эту ссылку в браузере (рекомендуется включить режим мобильного устройства в DevTools — F12 → Toggle Device Toolbar → выбрать iPhone или аналогичное устройство 390×844px).

## Сервисный слой (мок-данные)

Все данные находятся в файле `src/services/mockData.ts`. Функции организованы так, чтобы в будущем легко заменить мок-данные на реальные HTTP-запросы:

```typescript
// Пример замены мока на реальный API-вызов:

// Сейчас (мок):
export function getDeficits(): Deficit[] {
  return mockDeficits;
}

// После подключения бэкенда:
export async function getDeficits(): Promise<Deficit[]> {
  const response = await fetch('http://localhost:8080/api/deficits');
  return response.json();
}
```

### Базовый URL бэкенда

По умолчанию в мок-данных прописан базовый URL: `http://localhost:8080/`

При подключении реального бэкенда замените мок-функции в `src/services/mockData.ts` на `fetch`-вызовы к соответствующим эндпоинтам.

## API-эндпоинты (для будущей интеграции)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/products/search?name={query}` | Поиск продукта по наименованию |
| POST | `/api/meals` | Сохранение приёма пищи |
| GET | `/api/deficits?userId={id}&date={date}` | Получение дефицитов за день |
| GET | `/api/shopping-list?userId={id}` | Получение списка покупок |
| GET | `/api/recommendations?userId={id}` | Получение рекомендаций |

Подробная спецификация описана в `API_SPEC.md` (если создан) и в комментариях в `src/services/mockData.ts`.
